# Slack Notification Examples

This document shows examples of the daily Slack digests generated by the automated workflow.

## Example 1: Daily Digest With New Tooling Features

**Trigger:** The previous calendar day added notable entries to `CHANGELOG.md`.

**Daily Changelog Additions:**
```markdown
### Added
- Added Copilot CLI skill symlinks so sparkdock-managed skills are visible in `~/.copilot/skills/`
- Added `sjust sf-agents-refresh` and `sjust sf-agents-status` recipes

### Changed
- Changed Slack notifications to run once per day and summarize the previous day's meaningful updates
```

**Generated Notification:**
```
┌─────────────────────────────────────────────────────────┐
│ What shipped yesterday in Sparkdock                    │
├─────────────────────────────────────────────────────────┤
│                                                         │
│ Sparkdock shipped a small set of workflow improvements │
│ for AI-assisted tooling and release visibility.        │
│                                                         │
│ • Copilot CLI can now discover sparkdock-managed      │
│   skills automatically, which keeps shared AI         │
│   capabilities consistent across supported agents.    │
│ • New `sf-agents-*` recipes make it easier to refresh │
│   and inspect managed skills from the command line.   │
│ • Slack updates are now summarized once per day,      │
│   making project changes easier to scan at a glance.  │

│ Date: 2026-03-11 | Commits: b1c5812, a2d462f, 9fd8d80  │

```

## Slack Block Structure

The notifications use Slack's Block Kit for rich formatting:

```json
{
  "text": "What shipped yesterday in Sparkdock",
  "blocks": [
    {
      "type": "header",
      "text": {
        "type": "plain_text",
        "text": "What shipped yesterday in Sparkdock",
        "emoji": true
      }
    },
    {
      "type": "section",
      "text": {
        "type": "mrkdwn",
        "text": "<Claude's generated digest>"
      }
    },
    {
      "type": "context",
      "elements": [
        {
          "type": "mrkdwn",
          "text": "*Date:* 2026-03-11  |  *Commits:* <https://github.com/sparkfabrik/sparkdock/commit/abc123|abc123>, <https://github.com/sparkfabrik/sparkdock/commit/def456|def456>"
        }
      ]
    }
  ]
}
```

## Message Generation Guidelines

Claude is instructed to:
- Keep messages concise and scannable in Slack
- Focus on user-facing benefits, not implementation details
- Use a professional, informative tone
- Highlight the most impactful daily additions only
- Skip the digest entirely when the changes are not meaningful enough

## Notification Decision Logic

The workflow sends notifications when the previous calendar day contains:
- New significant user-facing features
- Major enhancements to existing functionality
- New commands or capabilities available to users
- Important integrations or tool additions

The workflow skips notifications for:
- Minor bug fixes
- Internal refactoring
- Documentation-only changes
- Build or CI configuration changes
- Dependency updates that do not unlock a new capability
- Days with no net additions in `## [Unreleased]`

## Testing Notifications

### Dry Run (No API Keys Required)

Validate the script without calling external APIs:

```bash
python3 src/slack-notify/notify-slack-on-merge.py --dry-run
```

### Offline Extraction Tests (No API Keys Required)

Validate the daily changelog extraction heuristics:

```bash
python3 src/slack-notify/notify-slack-on-merge.py --test
```

### Preview Mode (Requires `ANTHROPIC_API_KEY`)

Generate a real Claude digest for a specific day without posting to Slack:

```bash
export ANTHROPIC_API_KEY="your-key"
python3 src/slack-notify/notify-slack-on-merge.py daily --date 2026-03-11 --preview --ref origin/master
```

### Send Mode (Requires Both API Keys)

Send the digest for yesterday or replay a specific day:

```bash
export ANTHROPIC_API_KEY="your-key"
export SLACK_WEBHOOK_URL="your-webhook-url"

# Default: summarize yesterday in Europe/Rome
python3 src/slack-notify/notify-slack-on-merge.py daily --ref origin/master

# Replay a specific date
python3 src/slack-notify/notify-slack-on-merge.py daily --date 2026-03-11 --ref origin/master
```

## How Daily Changelog Analysis Works

The script compares the changelog snapshot immediately before the target day starts with the snapshot immediately before the next day starts.

This captures the final net additions to `## [Unreleased]` for that day, even if multiple commits touched the changelog. The workflow analyzes `origin/master` by default so manual replays stay aligned with shipped history. Claude then receives a structured list grouped by section, for example:

```markdown
### Added
- Added scheduled daily digest notifications

### Changed
- Changed Slack workflow to run once per day at 10:30 Europe/Rome
```

Claude only reasons about those extracted entries and decides whether they are meaningful enough to announce.
